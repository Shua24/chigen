# Internationalization (i18n) Overview - General translation management

Chigen supports **English** and **Bahasa Indonesia** with runtime language switching. No restart required.

## Layers

### 1. TranslationService (`Chigen.Core/Services/TranslationService.cs`)

Static class holding two `Dictionary<string, string>` dictionaries (`_en`, `_id`).

**Key methods:**
- `TranslationService.GetString(key)` — returns the translated string for the current language
- `TranslationService.CurrentLanguage = "en"|"id"` — switches language at runtime, fires `LanguageChanged` event

**Key scope naming conventions:**
| Prefix | Scope |
|--------|-------|
| `MainXxx`, `StatusXxx` | MainWindow labels & status messages |
| `SettingsXxx` | SettingsWindow labels |
| `PatientInfoXxx` | PatientInfoWindow labels |
| `HotkeyXxx` | HotkeySettingsWindow labels |
| `ConclusionXxx`, `RecommendationsXxx` | Dialog windows |
| `SaveXxx`, `DocxXxx`, `PdfXxx` | Export dialogs |
| `ShowPatientXxx` | Patient field toggles |
| `report_Xxx` | Document generator labels (patient info fields) |
| `SpecimenTypeXxx` | Sample type names |
| `ColXxx` | Table column headers |
| `LogoXxx` | Logo placement options |
| `XxxLabel` | Label suffixes for report fields |

### 2. Translations (`Chigen.App/Translations.cs`)

Non-static class implementing `INotifyPropertyChanged`. Registered as a global resource in `App.xaml`:

```xml
<local:Translations x:Key="Translations"/>
```

- One read-only property per translation key
- Subscribes to `TranslationService.LanguageChanged` → fires `PropertyChanged` for **all** properties
- WPF bindings automatically refresh because `PropertyChanged` is raised

**Binding pattern in XAML:**
```xml
<TextBlock Text="{Binding Mode=OneWay, Source={StaticResource Translations}, Path=MainTitle}"/>
```

`Mode=OneWay` is required because `Run.Text` defaults to `TwoWay` and the translation properties are read-only.

### 3. Document Generators (`Chigen.DocumentExport/*.cs`)

Both `DocxGenerator` and `DirectPdfGenerator` call `TranslationService.GetString()` directly (no WPF binding needed). This covers:

- Report title
- Patient info field labels
- Column headers (Cell Type, Count, %, Ref. Range)
- TOTAL row label
- Conclusion / Recommendations headings
- Footer labels (Report generated, Pathologist Signature)
- Default "No abnormalities detected." text
- Specimen type labels (Peripheral Blood / Bone Marrow Aspirate)

### 4. Language Persistence

`TemplateService` handles save/load:
- `TemplateService.LoadLanguage()` — reads from `%LOCALAPPDATA%\Chigen\language.json`
- `TemplateService.SaveLanguage(lang)` — persists the choice
- `App.OnStartup()` restores the saved language

The Settings window language ComboBox switches the language immediately via `TranslationService.CurrentLanguage`. On "Cancel", the language reverts to the persisted value.

## Adding a New Translation

1. **Add the key** to both `_en` and `_id` dictionaries in `TranslationService.cs`
2. **If used in XAML:** add a property to `Translations.cs`
3. **Bind** in XAML: `{Binding Mode=OneWay, Source={StaticResource Translations}, Path=PropertyName}`
4. **If used in code:** call `TranslationService.GetString("key")`
5. **Build and verify**

## Bug Fixes Applied

| Issue | Fix |
|-------|-----|
| Counter cells not visible on startup | Added `ApplySavedHotkeys()` + `RefreshEntries()` to `CounterViewModel` constructor |
| Patient badge hidden | Restored `_hasPatientInfo = true` default and `RefreshTemplateSettings()` call |
| PDF export crash ("Visible is not a method") | Changed `InvokeWordMethod(word, "Visible")` to `InvokeMember(…, SetProperty, …)` in `PdfConverter.cs` |
| Letterhead dropdown not translated | Made `LogoPlacementOptions` an instance property using `TranslationService.GetString()` |
| Run.Text bindings crash | Added `Mode=OneWay` to all `{Binding Source={StaticResource Translations}}" |
